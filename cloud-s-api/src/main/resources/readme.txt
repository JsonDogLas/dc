开发需求：
  云签API：为第三方提供 签名接口，返回签名页面
	
讨论结果
  1.接口安全，必须要通过授权，授权走那个 oauth2(智慧平台客户端授权模式)授权服务器授权	
  2.接口模式：返回签名页面为主，签名后提供签名后的结果或
    文件下载连接，还有这个签名页面的印章信息
  3.印章来源：
    1.页面上传（不对外开放）
    2.第三方自己提供
    3.电子文件平台的印章（与key关联的印章）
	

API 为 第三方提供的 接口
  1.授权（oauth2自带）		
  2.上传文件获取签名页面
  3.打开签名文件
  4.获取签名后的信息

云签API主要思路
  技术：oauth2,spring boot常用的框架
  思路
	签名池
	    1.正在签名的数据（SignContext）存在于内存中 通过上传文件获取签名页面 生成 signSessionId
		活跃期：15分钟 存在于内存(服务所在的内存)中的时间
		存活期：（1月）存在于 redis（部署在其他的服务器）
            2.将签名加载数据过程中 耗时的操作 做 异步线程处理
		1.转pdf，转图片 （异步线程处理）
		2.印章来源：（异步线程处理）
                    自定义印章（日期章，二维码），第三方加载，电子文件服务加载签章，。。。
		备注：上传文件后获取签名页面 查看线程有没有 执行完成，如果没有执行完成，等待结果后返回。（上传签名）		
	    3.定时器清理内存
                1.执行完的线程 填充到 SignContext
                2.15分钟 超时 就会清除 内存中 签名上下文
